Java版坦克大战

2019-03-08
Java, 练习

简介

此游戏是为了复习Java基础知识而写的，包含图形版和图片版。参考了马士兵的视频教程，在其基础上进行调整优化，并增加了一些新功能，例如：

  • 调整血块的掉落和效果规则
  • 增加技能条，优化炮弹的发射规则
  • 调整复活规则
  • 增加计分、计时功能
  • 增加暂停功能
  • 增加游戏结束画面
  • 增加重新开始功能
  • 优化图片素材

游戏玩法

  • 方向键控制坦克移动
    左：← 右：→ 上：↑ 下：↓ 左上：←↑ 左下：←↓ 右上：→↑ 右下：→↓
  • 普通炮弹：F键，朝当前运动方向发射，无限制
  • 超级炮弹：S键，同时向八个方向发射炮弹，消耗一定技能值
  • 复活：F2键，在随机地点复活并恢复全部血量
  • 暂停/继续游戏：空格键
  • 重新开始：R键

通过击杀敌方获得积分，死亡可消耗积分复活
玩家要在规定时间内取得尽可能高的分数

主要功能

坦克

敌方

  • 初始坐标随机
  • 在窗口范围内随机移动和开炮，可穿墙
  • 坦克之间发生碰撞后会改变方向
  • 被击中后产生爆炸并直接死亡，有概率掉落血块
  • 每当数量为零时自动生成，坐标随机

我方

  • 初始坐标随机
  • 在窗口范围内由方向键控制移动方向，可穿墙
  • 坦克上方显示血条和技能条
  • 按F键发射普通炮弹
  • 按S键消耗一定技能值发射超级炮弹，技能值为零时无法发射
  • 击杀敌人后得分
  • 被击中后产生爆炸，血量降低，血量为零时坦克死亡
  • 按F2键可以消耗积分在随机地点满血复活

子弹

无法穿墙，只能对不同阵营的坦克造成伤害

血块

能恢复部分血量和技能值，回满时不再生效
被吃掉或存在一段时间后自动消失

墙

挡子弹，作为掩护

爆炸

坦克被击中时产生，分不同阶段

计时器

显示倒计时，时间为零时游戏结束

窗口

  • 展现游戏画面，显示倒计时和当前得分
  • 按空格键暂停/恢复游戏
  • 游戏结束时显示得分，按R键重新开始
  • 大小固定，按“关闭”退出游戏

配置文件

可对游戏中的某些变量进行修改，如敌方数量、移动速度等

部分思路

动画效果

开启一个线程，循环执行run()方法里的内容。每隔一段时间就画出窗口内的所有元素（坦克、子弹、墙等），通过元素的增减和坐标变化实现动画效果

操作控制

使用键盘监听接收玩家指令，根据不同键值对坦克行为和游戏运行进行控制

碰撞检测

获取和该物体大小坐标相同的矩形，调用intersects()方法检测矩形是否重叠

游戏暂停/恢复

一开始是想让线程暂停，尝试后发现恢复不了
之后换了个方法：用if else语句将线程执行的内容分为游戏状态和暂停状态，按空格键在两种状态间切换，但是线程一直运行

重新开始

定义一个init()方法，用于初始化
当游戏结束后，显示结束画面，提示玩家可以按R键重新开始，按下该键后执行init()方法，所有数据恢复到初始状态

遇到的问题

屏幕闪烁

原因：重画的频率太快，paint()方法来不及画出本次界面全部内容就被刷新了
解决方法：使用双缓冲，将所有东西画在一个虚拟图像上，再用pain()一次性显示出来

图片版第一辆坦克被击中后无爆炸效果

原因：用的是Toolkit.getImage()，延迟加载，第一次爆炸的时候图片还没加载，所以无法正常显示
解决方法：
1.还是用Toolkit.getImage()，第一次爆炸前先让它在看不到的地方爆炸一次，使图片加载进来（马士兵的方法）
2.ImageIcon.getImage()方法
3.ImageIO.read()方法
注意文件路径

使用ImageIO.read()时又遇到了一些问题，以加载某一张爆炸图片为例：
Image image = ImageIO.read(Explosion.class.getClassLoader().getResource(“images/10.gif”));//第一辆不爆炸
Image image = ImageIO.read(new File(“images/10.gif”));//报错
Image image = ImageIO.read(new File(“src/images/10.gif”));//第一辆正常爆炸
大概还是延迟加载和路径问题吧……

图片版游戏开始时坦克闪烁，之后正常

原因：使用了Toolkit.getImage()
解决方法：用另外两种加载方式

坦克初始化时如果坐标重叠就会卡住，无法移动

原因：坦克类里的collidesWithTanks()方法使两辆坦克相撞时返回相撞前坐标，避免重叠
初始化时无旧坐标，如果坐标重叠只能停留在原地
解决方法：
没有什么是按F解决不了的，有就按S，再不行就按F2
（要正经解决的话，应该加个循环判断吧，如果当前对象的坐标和已有坦克重叠，就生成新的随机坐标，直到无重叠）
