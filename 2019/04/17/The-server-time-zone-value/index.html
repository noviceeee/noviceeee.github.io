The server time zone value '???ú±ê×??±??' is unrecognized or represents more than one time zone

2019-04-17
MySQL

报错信息：

SQLException: The server time zone value ‘???ú±ê×??±??’ is unrecognized or represents more than one time zone. You must configure either the server or JDBC driver (via the serverTimezone configuration property) to use a more specifc time zone value if you want to utilize time zone support.

解决方法：

方法一：指定时区

serverTimeZone时区常用值
UTC: 协调世界时，又称世界统一时间、世界标准时间、国际协调时间。由于英文（CUT）和法文（TUC）的缩写不同，作为妥协，简称UTC。中国时间与UTC的时差为+8，也就是UTC+8。
GMT%2B8: 中国时间(GMT+8这种方式支持全部时区，通过修改最后的数字实现，不过%2B不能写成+)
Asia/Shanghai：上海时间

在数据库连接语句后添加?serverTimezone=Asia/Shanghai



conn = DriverManager.getConnection("jdbc:mysql://localhost/mydata?serverTimezone=Asia/Shanghai&user=root&password=root");
或
conn = DriverManager.getConnection("jdbc:mysql://localhost/mydata?serverTimezone=Asia/Shanghai","root","root");
都行

方法二：在MySQL中设置时区（默认为SYSTEM）
set global time_zone = ‘+8:00’; 修改mysql全局时区为北京时间，即我们所在的东8区
set time_zone = ‘+8:00’; 修改当前会话时区
